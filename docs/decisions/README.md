## 개요

아키텍처 결정 기록(Architecture Decision Records, ADR)은 프로젝트의 주요 기술적 결정을 문서화합니다. 각 ADR은 결정의 컨텍스트, 고려한 대안, 최종 결정 및 그 결과를 기록합니다.

## ADR 작성 가이드

### 새 ADR 작성 시

1. 다음 번호로 파일 생성: `XXX-짧은-제목.md`
2. 아래 템플릿 사용
3. 이 README의 목록에 추가
4. PR로 팀 리뷰 후 머지

### 상태 정의

| 상태 | 설명 |
|------|------|
| **제안됨** | 리뷰 대기 중 |
| **수락됨** | 채택되어 적용 중 |
| **수락됨 (임시)** | 현재 적용 중이나 향후 변경 예정 |
| **대체됨** | 새로운 ADR로 대체됨 |
| **폐기됨** | 채택하지 않기로 결정 |

### ADR 템플릿

```markdown
# ADR-XXX: 제목

> **상태**: 제안됨 | 수락됨 | 대체됨 | 폐기됨
> **날짜**: YYYY-MM
> **관련 ADR**: (있는 경우)
```
